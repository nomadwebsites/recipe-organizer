═══════════════════════════════════════════════════════════════════
                        HOW TO USE
═══════════════════════════════════════════════════════════════════

METHOD 1: Import from URL (Best for Budget Bytes, food blogs)
  1. Click "Import from URL"
  2. Paste recipe URL
  3. Click "Import Recipe"
  4. Wait 2-5 seconds
  5. Review and save

METHOD 2: 📋 Paste Recipe Text (Works with ANY site!)
  1. Open recipe on ANY website
  2. Select all (Ctrl+A) and copy (Ctrl+C)
  3. Click "📋 Paste Recipe" in app
  4. Paste (Ctrl+V)
  5. Click "Parse Recipe"
  6. Review and save

METHOD 3: Manual Entry
  1. Click "Add Recipe"
  2. Fill in the form
  3. Save

SEARCH:
  - Type ingredient, recipe name, or tag
  - Press Enter
  - Instant results!

  - Schema: recipes table + FTS index
  - Location: /opt/recipe-organizer/recipes.db

FRONTEND:
  - HTML5 (semantic markup)
  - CSS3 (modern styling, flexbox, grid)
  - Vanilla JavaScript (no frameworks!)
  - Responsive design (mobile-first)

DEPLOYMENT:
  - Systemd service (auto-start on boot)
  - Gunicorn WSGI server (3 workers)
  - Nginx reverse proxy (optional)
  - UFW firewall configuration

API ENDPOINTS:
  GET    /api/recipes          - List all recipes
  GET    /api/recipes/<id>     - Get one recipe
  POST   /api/recipes          - Create recipe
  PUT    /api/recipes/<id>     - Update recipe
  DELETE /api/recipes/<id>     - Delete recipe
  GET    /api/search?q=<query> - Search recipes
  POST   /api/parse-url        - Parse from URL
  POST   /api/parse-text       - Parse from text
  GET    /api/health           - Health check

═══════════════════════════════════════════════════════════════════
                         TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════

❓ Can't import from URL?
   → Try "📋 Paste Recipe" feature instead!

❓ Service won't start?
   → Check logs: sudo journalctl -u recipe-organizer -n 50

❓ API key error?
   → Verify .env file: cat /opt/recipe-organizer/.env

❓ Port already in use?
   → Change PORT in .env or kill process on 5000

❓ Can't access from another device?
   → Check firewall: sudo ufw allow 5000/tcp

❓ Nginx won't start?
   → Test config: sudo nginx -t
   → Check logs: sudo journalctl -u nginx
